     */

    **Algorithm (per D-09, D-10):**
    1. Filter to only pending/scheduled matches that have `team1_id` or `team2_id` AND `court_id` AND `scheduled_date` AND `scheduled_time`
    2. Build a map: `teamId -> [{ matchId, courtId, date, time, durationMinutes }]`
       - For each match, add entries for both team1_id and team2_id (if non-null)
    3. For each team with 2+ matches, check all pairs:
       - Skip pairs on the same court (same-court matches are sequential)
       - Skip pairs on different dates (no overlap possible)
       - For same-date pairs on different courts: compute time ranges
         - `startA = parseTime(timeA)`, `endA = startA + durationA`
         - `startB = parseTime(timeB)`, `endB = startB + durationB`
         - Overlap if `startA < endB && startB < endA`
       - If overlap, add to conflicts array
    4. Deduplicate: avoid reporting same pair twice (sort match IDs, use Set of `matchA-matchB` keys)
    5. Return conflicts array

    **parseTime helper:** Reimplement locally (same as cascadeRecalculator.js):
    ```javascript
    function parseTime(timeStr) {
      if (!timeStr) return 0
      const [h, m] = timeStr.split(':').map(Number)
      return h * 60 + m
    }
    ```
