    **Signature:**
    ```javascript
    export async function applyCascadeOnResume(supabaseClient, tournamentId, courtId)
    ```

    **Return type:**
    ```javascript
    { success: boolean, updatedCount: number, spillOver: boolean, spillOverDate?: string, error?: string }
    ```

    **Implementation steps (per D-02, D-04):**
    1. Construct anchor time: `const anchorTime = new Date().toISOString()` (per D-04)
    2. Fetch court constraints from `courts` table: `SELECT id, name, available_from, available_to, break_start, break_end WHERE id = courtId` — NOTE: fetch `name` too, needed for per-player notification message
    3. Fetch ALL matches on this court for this tournament: `SELECT id, match_number, court_id, scheduled_date, scheduled_time, estimated_duration_minutes, status, team1_id, team2_id, phase FROM tournament_matches WHERE tournament_id = tournamentId AND court_id = courtId ORDER BY scheduled_date, scheduled_time, match_number`
    4. Fetch tournament dates: `SELECT start_date, end_date FROM tournaments WHERE id = tournamentId`
    5. Generate `tournamentDays` array from start_date to end_date (same loop as existing function)
    6. Call `recalculateCourt({ courtId, actualEndTime: anchorTime, courtMatches, court, tournamentDays })`
    7. Check spill-over: `const spillOver = updates.some(u => u.scheduled_date > tournament.end_date)`. If spillOver, set `spillOverDate` to the max scheduled_date in updates.
    8. Persist updates: batch UPDATE each match's `scheduled_date` and `scheduled_time` (same loop pattern as existing function)
    9. Send per-player resume notifications (per D-13, D-14, D-15):
       - Collect unique team IDs from pending matches on this court (non-null team1_id/team2_id from matches with status != 'completed')
       - Fetch registrations: `SELECT id, player1_id, player2_id FROM tournament_registrations WHERE id IN (uniqueTeamIds)`
       - For each player, find their first upcoming pending match on this court (the match with the earliest scheduled_date + scheduled_time where status != 'completed' and (team1_id = regId OR team2_id = regId))
       - Build notification per player with `type: 'schedule_change'` and message per D-14: `La cancha ${courtName} ha sido reanudada. Tu proximo partido: ${nextMatchTime} en ${courtName}.` where `nextMatchTime` is formatted as "HH:MM" from the player's first upcoming match
       - Call `createBulkNotifications(supabaseClient, notifications)`
    10. Return `{ success: true, updatedCount: updates.length, spillOver, spillOverDate }`

    **Also add tests** to `src/lib/cascadeRecalculator.test.mjs`:
    - Test "resume anchor — cascade starts from current time": Create matches where m1 at 09:00 is completed, m2 at 10:00 is scheduled, m3 at 11:00 is scheduled. Call recalculateCourt with actualEndTime = "2025-06-15T15:00:00" (simulating resume at 15:00). Expect m2 at 15:00 and m3 at 16:00.
    - Test "resume anchor — no triggering completed match uses anchor directly": All matches are pending. Call with actualEndTime = "2025-06-15T14:00:00". Expect first pending match at 14:00.
