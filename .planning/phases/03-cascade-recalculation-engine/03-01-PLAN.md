<objective>
Build the cascade recalculation engine and its DB persistence layer. When a match on a court finishes (has actual_end_time), all remaining pending/scheduled matches on that same court are recalculated in sequence: next match starts at actual end time, each subsequent match starts at previous start + estimated_duration_minutes. Break windows and day overflow are handled. Only time fields change; everything else is preserved.

Purpose: Core engine that Phase 4 will wire into the save flows.
Output: `cascadeRecalculator.js` (pure logic), `cascadeRecalculator.test.js` (unit tests), `cascadeSchedulePersistence.js` (DB layer)
</objective>

    ```javascript
    /**
     * Recalculates scheduled times for pending matches on a specific court
     * after a match finishes with a known actual end time.
     *
     * @param {Object} params
     * @param {string} params.courtId - The court where recalculation happens
     * @param {string} params.actualEndTime - ISO timestamp of when the completed match ended (e.g. "2025-06-15T14:30:00")
     * @param {Array} params.courtMatches - ALL matches on this court (completed + pending), each with:
     *   { id, match_number, court_id, scheduled_date, scheduled_time, estimated_duration_minutes, status, team1_id, team2_id, phase }
     * @param {Object} params.court - Court constraints: { available_from: "HH:MM", available_to: "HH:MM", break_start: "HH:MM"|null, break_end: "HH:MM"|null }
     * @param {Array} params.tournamentDays - Ordered array of tournament day strings ["2025-06-15", "2025-06-16", ...]
     * @returns {Array} updates - Array of { id, scheduled_date, scheduled_time } for matches that changed. Empty if nothing changed.
     */
    export function recalculateCourt({ courtId, actualEndTime, courtMatches, court, tournamentDays }) {
    ```

    Algorithm:
    1. Filter `courtMatches` to only this `courtId` (defensive — caller should already filter, but enforce ISO-01).
    2. Separate into completed (status === 'completed') and pending (status === 'scheduled' or status === 'pending').
    3. Sort pending by `scheduled_date` ASC, then `scheduled_time` ASC, then `match_number` ASC.
    4. Parse `actualEndTime` to extract the date ("YYYY-MM-DD") and time ("HH:MM") — this is the cascade anchor.
    5. Filter pending to only matches that are on the same day as the anchor AND have `scheduled_time >= anchor_time`, OR are on days after the anchor date. These are the "affected" matches.
    6. Starting from the anchor time, assign each affected match a new start time:
       a. `cursor = { date: anchorDate, time: anchorTimeMinutes }`
       b. For each match in order:
          - Check if `cursor.time` falls inside break window (cursor.time >= break_start_minutes AND cursor.time < break_end_minutes). If so, set `cursor.time = break_end_minutes`.
          - Check if `cursor.time + match.estimated_duration_minutes > available_to_minutes`. If so, overflow to next tournament day:
            - Find next day in `tournamentDays` after `cursor.date`.
            - If no next day exists, keep the match on the last day at available_from (best effort).
            - Set `cursor = { date: nextDay, time: available_from_minutes }`.
            - Re-check break window on the new day (if available_from falls in break, push to break_end).
          - Also check: if `cursor.time` itself exceeds `available_to_minutes` (match can't even start), overflow to next day.
          - Assign `newDate = cursor.date`, `newTime = minutesToTime(cursor.time)`.
          - Advance cursor: `cursor.time += match.estimated_duration_minutes`.
       c. If `newDate !== match.scheduled_date` or `newTime !== match.scheduled_time`, add to updates array.
    7. Return updates array: `[{ id: match.id, scheduled_date: newDate, scheduled_time: newTime }]`.

    IMPORTANT constraints per SCHED-09 / ISO-02: The function MUST NOT modify or return changes to: `court_id`, `team1_id`, `team2_id`, `phase`, `match_number`, `status`, `estimated_duration_minutes`. Only `id`, `scheduled_date`, `scheduled_time` in the return.

    Implementation steps:

    1. **Fetch the completed match** to get its `court_id` and `actual_end_time`:
       ```javascript
       const { data: completedMatch, error: matchError } = await supabaseClient
         .from('tournament_matches')
         .select('id, court_id, actual_end_time')
         .eq('id', matchId)
         .single()
       ```
       If `!completedMatch` or `!completedMatch.actual_end_time` or `!completedMatch.court_id`, return `{ success: true, updatedCount: 0 }` (nothing to recalculate — e.g., elimination match without court).

    2. **Fetch the court** constraints:
       ```javascript
       const { data: court, error: courtError } = await supabaseClient
         .from('courts')
         .select('id, available_from, available_to, break_start, break_end')
         .eq('id', completedMatch.court_id)
         .single()
       ```
       If error, return `{ success: false, updatedCount: 0, error: courtError.message }`.

    3. **Fetch ALL matches on this court** for this tournament (completed + pending, all days):
       ```javascript
       const { data: courtMatches, error: cmError } = await supabaseClient
         .from('tournament_matches')
         .select('id, match_number, court_id, scheduled_date, scheduled_time, estimated_duration_minutes, status, team1_id, team2_id, phase')
         .eq('tournament_id', tournamentId)
         .eq('court_id', completedMatch.court_id)
         .order('scheduled_date')
         .order('scheduled_time')
         .order('match_number')
       ```

    4. **Fetch tournament date range** for `tournamentDays` array:
       ```javascript
       const { data: tournament, error: tError } = await supabaseClient
         .from('tournaments')
         .select('start_date, end_date')
         .eq('id', tournamentId)
         .single()
       ```
       Generate `tournamentDays` array from `start_date` to `end_date` inclusive (same date iteration pattern used in ScoreboardPage lines 84-96):
       ```javascript
       const tournamentDays = []
       const start = new Date(tournament.start_date + 'T00:00:00')
       const end = new Date(tournament.end_date + 'T00:00:00')
       for (let d = new Date(start); d <= end; d.setDate(d.getDate() + 1)) {
         const yyyy = d.getFullYear()
         const mm = String(d.getMonth() + 1).padStart(2, '0')
         const dd = String(d.getDate()).padStart(2, '0')
         tournamentDays.push(`${yyyy}-${mm}-${dd}`)
       }
       ```

    5. **Call the cascade engine**:
       ```javascript
       const updates = recalculateCourt({
         courtId: completedMatch.court_id,
         actualEndTime: completedMatch.actual_end_time,
         courtMatches,
         court,
         tournamentDays,
       })
       ```

    6. **Persist updates** — batch UPDATE each match's `scheduled_date` and `scheduled_time`. Use individual updates in a loop (Supabase JS client doesn't support batch update by different IDs in one call):
       ```javascript
       if (updates.length === 0) return { success: true, updatedCount: 0 }

       for (const update of updates) {
         const { error: updateError } = await supabaseClient
           .from('tournament_matches')
           .update({
             scheduled_date: update.scheduled_date,
             scheduled_time: update.scheduled_time,
           })
           .eq('id', update.id)

         if (updateError) {
           return { success: false, updatedCount: 0, error: `Failed to update match ${update.id}: ${updateError.message}` }
         }
       }

       return { success: true, updatedCount: updates.length }
       ```

    Error handling: wrap the entire function body in try/catch, return `{ success: false, updatedCount: 0, error: err.message }` on any exception.

    IMPORTANT per ISO-01: The function only queries matches for `completedMatch.court_id`. Other courts are never touched.
    IMPORTANT per ISO-02: The UPDATE only sets `scheduled_date` and `scheduled_time` — no other fields.
